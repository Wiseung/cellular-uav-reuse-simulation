## A. 任务目标重述

**目标重述（用我的话）**
在一个规则蜂窝网络里，研究"频率复用因子 N"这个设计旋钮如何决定同频干扰的强弱，进而决定用户的 SIR（信干比）和整个系统的容量/频谱效率；并在此基础上引入"低空用户（如无人机 UAV）"，说明它相比地面用户为什么干扰特性不同、SIR 会变差还是变好，最终给出对低空融合组网频率规划的工程启示。

**本课题最终要回答的核心问题**
1. N 增大时，同频干扰如何变化，边缘用户 SIR 如何变化？（定量：SIR-N 曲线）
2. N 增大在提升 SIR 的同时会牺牲什么？系统容量/频谱效率随 N 如何变化，存在怎样的折中？（定量：容量-N 曲线，找折中点）
3. 同样的网络里，低空用户和地面用户的 SIR/容量差异有多大，为什么？（关键物理：低空用户对多个基站呈 LOS，接收到更多强干扰）
4. 上述结论对"蜂窝+低空融合网络/6G 低空频率规划"意味着什么？

**成功标准（可验证）**
- [ ] 有一条**解析基线**：闭式公式 SIR(N, n) 能算出经典数值（如 N=7、路损指数 n=4 时边缘 SIR≈18.6 dB），并被仿真复现（两者误差可解释）。
- [ ] 至少产出 **3 组对照实验**：扫 N、扫 UAV 高度、扫路损指数/LOS-NLOS。
- [ ] 至少 **6–8 张规范图 + 2 张参数/结果表**，每张图都能对应报告里一段分析。
- [ ] 结论里明确写出**干扰-复用-容量三者折中**，并落到低空组网工程意义。
- [ ] 报告 ≥3000 字，结构齐全，代码可一键复现（固定随机种子）。
- [ ] 无任何虚构文献条目。

## B. 技术路线选择

| 路线 | 做法 | 优点 | 缺点/工作量 | 拿分稳定性 |
|---|---|---|---|---|
| **路线1：解析几何模型（教科书法）** | 规则六边形，边缘用户，第一层 6 个同频干扰，闭式 SIR=(3N)^{n/2}/6 | 极简、可解释、有权威闭式解、几乎不会错 | 只有确定性曲线，图偏少，无分布、无随机性，UAV 只能靠改 n 体现 | 高，但单独用略单薄 |
| **路线2：轻量 Monte Carlo（推荐主线）** | 六边形网格布站，用户随机撒点，计算每个用户的经验 SIR，出 CDF/均值/边缘分位数；UAV 用"高度→LOS 概率/路损指数"体现 | 图丰富（CDF、箱线、热力），能自然对比地面 vs 低空，能出边缘用户统计 | 需要写布站+距离+采样，代码量中等；参数要设合理 | **最高**（内容足、故事完整） |
| **路线3：随机几何/3D射线追踪/PPP+ITU空地模型** | 泊松撒站、3D、精细 LOS 概率、可能 GPU | 前沿、上限高 | 复杂、易翻车、超出课程范围、调不出来会拖垮进度 | 低（风险高，性价比差） |

**推荐主路线：路线2（Monte Carlo）为主 + 路线1（解析）为基线校验。**

**为什么最适合课程大作业而非科研论文：**
- 路线1 提供"理论锚点"，让报告的理论部分有权威闭式公式可写；路线2 提供"丰富图表"，让核心分析部分有料。两者互为验证（解析≈仿真）本身就是一个强有力的得分点——体现"理论与仿真结合"。
- 完全避开路线3 的复杂度陷阱，符合"最小可行但足够拿高分"。
- UAV 差异不靠昂贵的 3D 射线追踪，而靠"高度改变 LOS 比例→改变有效路损指数/干扰数量"这一**物理清晰的简化**来体现，既贴题又不失控。

## C. 理论模型框架

**核心理论及其在本题中的作用：**

1. **蜂窝频率复用原理**
   - 作用：解释为何要把频谱分成 N 组、同一组频率在相隔一定距离的小区重复使用；N 就是本题的主变量。

2. **复用距离与复用因子关系**
   - 关键公式：簇大小(复用因子) $N=i^2+ij+j^2$（合法值 1,3,4,7,9,12,13…）；同频复用比 $Q=D/R=\sqrt{3N}$。
   - 作用：把"抽象的 N"翻译成"几何上同频基站离得多远"，是连接 N 与干扰强度的桥梁。

3. **路径损耗模型（log-distance）**
   - 公式：$PL(d)=PL(d_0)+10\,n\,\log_{10}(d/d_0)$，接收功率 $P_r \propto P_t\,d^{-n}$。
   - 作用：决定期望信号和干扰信号随距离的衰减；n 是区分环境（城区/郊区）和地面/低空的关键。

4. **同频干扰建模**
   - 只考虑**第一层 6 个同频干扰小区**（主线；干扰随距离快速衰减，第一层贡献主导）。
   - 总干扰 $I=\sum_{k=1}^{6} P_t\,d_k^{-n}$。
   - 作用：本题"干扰"的具体来源与量化方式。

5. **SIR 定义与计算**
   - 定义（忽略噪声、干扰受限）：$\text{SIR}=\dfrac{P_r^{\text{desired}}}{\sum_k P_r^{(k)}}$。
   - 边缘用户闭式解：$\text{SIR}=\dfrac{1}{6}\left(\dfrac{D}{R}\right)^n=\dfrac{(3N)^{n/2}}{6}$，dB 值 $=5n\log_{10}(3N)-10\log_{10}6$。
   - 作用：本题第一大核心指标。

6. **系统容量 / 频谱效率的简化表征**（这是最容易"定义不清"的地方，必须写清楚）
   - 每小区信道数 $\propto 1/N$（总频谱固定，分成 N 组）。
   - **区域频谱效率(推荐主指标)**：$\text{ASE}=\dfrac{1}{N}\log_2(1+\text{SIR})$ [bit/s/Hz/cell]，同时体现"复用越激进(N小)可用带宽越多"与"SIR 越高单位带宽承载越高"的**对立折中**。
   - 作用：本题第二大核心指标，用来找"N 的甜点区"。

**地面用户 vs 低空用户 的建议建模方式（简化但物理清晰）：**
- **地面用户**：多为 NLOS，路损指数大（$n\approx 3.5\sim4$），干扰随距离衰减快，第一层干扰主导。
- **低空/UAV 用户**：飞得越高，对越多基站（含远处同频基站）呈 **LOS**，传播接近自由空间（$n\to 2$）。后果有二：①期望信号变强，②但**干扰也变强且干扰源变多**，净效果通常是 **SIR 下降**（这是本题最漂亮的物理结论）。
- **两种落地方式（都可，先用简单版）：**
  - **简单版(推荐)**：让路损指数随高度变化 $n(h)$，从地面 $n\approx3.8$ 平滑降到高空 $n\approx2.0$；同时干扰小区数从 6 扩展到第二层（因高空能"看到"更远同频站）。
  - **进阶版(可选)**：引入基于仰角的 LOS 概率 $P_{LOS}(\theta)$，对每条链路按概率取 LOS/NLOS 两套参数。**注意：LOS 概率模型的具体系数请去检索 3GPP TR 36.777 / ITU-R P.1410 空对地模型，不要凭空写系数。**

**合理简化假设（务必在报告中显式列出）：**
- 所有基站等发射功率、全向天线；忽略天线方向图与下倾。
- 忽略阴影衰落与小尺度衰落（主线），或仅作为可选扩展加对数正态阴影。
- 干扰受限系统，忽略热噪声（只分析 SIR）；SINR 作为可选。
- 只算下行；用户就近接入最强基站（或指定服务基站）。
- 六边形规则网络、基站位于小区中心。

## D. 仿真场景设计

**场景**：单频段宏蜂窝网络，规则六边形布站；在其上叠加地面用户与低空 UAV 用户两类终端，研究不同 N 下的下行 SIR 与频谱效率。

**网络拓扑设置建议：**
- 采用**规则六边形网络**，中心小区为研究对象，周围布 2 圈（19 站）或 3 圈以减小边界效应。
- 主线**只算第一层 6 个同频干扰小区**；低空/进阶情形再纳入第二层同频站。
- 基站在小区中心；地面用户在中心小区内**均匀随机撒点**（Monte Carlo，如 2000 点）；边缘用户用"到基站距离≈R"的点或取 SIR 的 5% 分位数。
- UAV 用户：水平位置同样随机撒点，**高度 h 作为扫描维度**；链路距离用 3D 距离 $\sqrt{d_{2D}^2+h^2}$。

**推荐参数表：**

| 参数 | 符号 | 推荐默认值 | 推荐扫描范围 |
|---|---|---|---|
| 载频 | $f_c$ | 2 GHz | {2, 3.5} GHz（可选） |
| 小区半径 | $R$ | 500 m | 300–1000 m（可选） |
| 基站发射功率 | $P_t$ | 46 dBm | 固定即可（SIR 中相消） |
| 噪声 | — | 忽略（仅分析 SIR） | 可选加 -95 dBm 做 SINR |
| **复用因子** | $N$ | 7 | **{1, 3, 4, 7, 12}** |
| UAV 高度 | $h$ | 100 m | **0(地面)–300 m**，步进 25–50 m |
| 地面路损指数 | $n_g$ | 3.8 | 3.5–4.0 |
| 高空路损指数 | $n_a$ | 2.2 | 2.0–2.5 |
| 干扰层数 | — | 第一层(6站) | 第一层→第二层(可选) |
| Monte Carlo 采样点 | — | 2000/场景 | 1000–5000 |
| 随机种子 | — | 固定(如 42) | — |

> 说明：因所有基站等功率，SIR 与 $P_t$、$P_r(d_0)$ 无关，故发射功率、载频对纯 SIR 分析不敏感——这点可在报告里点明，体现你理解"干扰受限"本质。

## E. 实验设计

**核心实验（必须完成，3 个）**

**实验1：复用因子 N 对边缘 SIR 的影响（主线中的主线）**
- 变量：N ∈ {1,3,4,7,12}
- 不变：地面用户、n=3.8、第一层干扰
- 输出：SIR(dB)-N 曲线（解析 vs 仿真两条叠加）；一张结果表
- 预期：N↑ → SIR 单调↑（N=7 附近 SIR≈18 dB 左右量级）
- 支撑：理论主体 + 核心分析（验证解析≈仿真）

**实验2：复用因子 N 对系统容量/频谱效率的影响（折中）**
- 变量：N ∈ {1,3,4,7,12}
- 不变：同实验1
- 输出：ASE=(1/N)·log2(1+SIR) vs N 曲线；同时画"每小区信道数∝1/N"辅助
- 预期：ASE 非单调或呈折中——小 N 频谱复用密但 SIR 低，大 N SIR 高但可用带宽少，存在甜点区
- 支撑：核心分析 + 结论（复用-容量折中）

**实验3：地面用户 vs 低空用户 SIR 对比（贴题关键）**
- 变量：UAV 高度 h（0→300 m）
- 不变：N=7、随高度变化的 n(h)、干扰随高度扩到第二层
- 输出：SIR vs 高度 曲线（地面为 h=0 基准）；地面/低空 SIR 的 CDF 对比
- 预期：h↑ → LOS 增多 → 干扰增强 → SIR 明显下降；低空用户 SIR CDF 整体左移
- 支撑：核心分析（低空干扰特性差异）+ 结论（低空组网启示）

**扩展实验（有余力再做，任选）**
- **扩展A**：不同路损指数 n（3.5/3.8/4.0）对 SIR-N 曲线的影响 → 环境依赖性。
- **扩展B**：LOS/NLOS 概率模型（进阶版）替代简单 n(h)。
- **扩展C**：加对数正态阴影，看 SIR CDF 展宽与边缘用户恶化。
- **扩展D**：SINR（加噪声）与 SIR 对比，说明干扰受限假设的适用性。

> 控制总量：核心 3 个必做，扩展最多挑 1–2 个，避免贪多。

## G. Python 实现计划

**推荐库**：numpy、matplotlib、pandas（可选 scipy 仅用于统计分位数，非必需）。

**目录结构：**
```
cellular_uav_sir/
├── config.py          # 所有参数（N候选、R、n、高度范围、种子）
├── geometry.py        # 六边形布站、同频干扰站坐标、复用距离D=√(3N)R
├── pathloss.py        # 路损模型 Pr∝d^-n；n(h) 高度相关；(可选)LOS概率
├── sir_analytic.py    # 解析闭式 SIR=(3N)^{n/2}/6
├── sir_montecarlo.py  # 随机撒点、算每用户SIR、返回均值/边缘分位/CDF数据
├── experiments.py     # 跑实验1/2/3，结果存 pandas DataFrame → CSV
├── plots.py           # 统一出图（复用matplotlib风格）
├── main.py            # 一键复现：跑全部核心实验并出图
└── results/           # 图(png) + 表(csv)
```

**各模块职责：**
- config：集中管理参数，固定随机种子（可复现）。
- geometry：给定 N 生成同频干扰基站的相对坐标与距离 D。
- pathloss：接收功率与距离/高度/指数的关系；高度→n(h) 映射。
- sir_analytic：闭式基线，一行公式，用来校验。
- sir_montecarlo：核心引擎，输出 SIR 数组供统计与画 CDF。
- experiments：编排 3 个核心实验，产出 DataFrame 与 CSV。
- plots：把 DataFrame 画成 F 节的图。

**迭代顺序（先跑通最小版本再扩展）：**
1. **v0 最小版**：只写 `sir_analytic`，打印 N∈{1,3,4,7,12}、n=4 的 SIR，核对 N=7≈18.6 dB。**先把理论跑对。**
2. **v1**：写 geometry + montecarlo，地面用户单一 N 的 SIR 均值，验证与解析同量级。
3. **v2**：扫 N，出图2、图3（实验1、2）。
4. **v3**：加高度维度与 n(h)，出图4、图5（实验3）。
5. **v4（可选）**：挑 1 个扩展实验。
6. **v5**：统一美化图表、导出 CSV、写 README/复现说明。

**最容易出错的点与自检：**
- **复用距离公式**：确认用的是 $D=\sqrt{3N}\,R$，别写成 $\sqrt{N}$。自检：N=7 → D/R≈4.58。
- **dB 与线性混用**：SIR 求和必须在**线性域**做（$\sum d_k^{-n}$），最后再转 dB。自检：先算线性 SIR 再 `10*log10`。
- **容量定义不清**：明确 ASE=(1/N)·log2(1+SIR_linear)，SIR 用线性值。自检：手算一组核对。
- **解析≠仿真差太多**：多半是干扰站距离/层数设置不一致，或用户位置不是边缘。自检：把仿真里的用户固定到小区边缘，应逼近解析值。
- **UAV 高度没进 3D 距离**：确认用 $\sqrt{d_{2D}^2+h^2}$。自检：h=0 时应退化回地面结果。
- **随机性导致不可复现**：固定 `np.random.seed`。

## L. 给学生的最终建议

**最该抓住的三件事：**
1. **抓主线**：把"复用因子 N → 干扰 → SIR → 容量折中"这条链讲透，用一条闭式公式 $\text{SIR}=(3N)^{n/2}/6$ 当理论锚点，再用仿真复现它——这是最稳的高分点。
2. **抓贴题差异**：地面 vs 低空的对比不能省。核心物理只有一句话，务必讲清楚：**UAV 飞得高→对更多基站呈 LOS→干扰变强变多→SIR 反而下降**。这句话就是你区别于"普通蜂窝作业"的加分项。
3. **抓图表与解读**：先保证 F 节 5 张必做图跑出来，每张图配 2–4 句"回到通信工程意义"的解读，报告的核心分析部分自然就满了。

**最不值得折腾的：**
- 不要碰 3D 射线追踪、随机几何 PPP、机器学习优化、大规模网络规划——性价比极低，容易翻车。
- 不要纠结精确的天线方向图、阴影/多径细节；主线明确假设"干扰受限、忽略噪声与衰落"即可，把它写成显式假设反而显专业。
- 不要在参考文献上冒险——宁可少引、只引查证过的真文献，也绝不编。

一句话：**理论一条公式、仿真复现它、地面对比低空、图配解读回到工程意义**——这题就稳了。
