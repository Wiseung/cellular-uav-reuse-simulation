# 答辩讲稿

下面这版讲稿按 `7-10` 分钟口头汇报准备，已经和当前分支的最新结果完全对齐。当前主线不再是“`SIR + 扰动蜂窝`”，而是“`SINR + 可配置天线 + 真实站址 + 动态实验`”。

## 一、开场
各位老师好，我的课程设计题目是《蜂窝 + 低空融合网络频率复用与 SINR 信干噪比仿真：不同复用因子、真实站址布局与动态低空用户场景下的干扰、频谱效率与链路性能分析》。

这次汇报我重点回答四个问题：

1. 复用因子变化时，地面用户的 SIR 和 SINR 如何变化；
2. 在更工程化的指标下，系统有效频谱效率如何变化；
3. 低空 UAV 用户的 SINR 随高度如何变化；
4. 如果接入真实站址并引入轨迹、切换和负载，低空链路的动态过程会发生什么变化。

和主分支相比，这一版模型最大的变化是三点：第一，核心指标从纯 SIR 升级成了同时计算 SIR 和 SINR；第二，增加了真实站址数据驱动的布局接口；第三，增加了动态 UAV 轨迹实验，不再只看静态快照。

## 二、研究背景
传统蜂窝网络里的频率复用，本质上是在“空间复用密度”和“同频干扰水平”之间做平衡。

如果复用因子小，比如 `N=1`，所有小区都使用同一组频谱，单位面积容量高，但同频干扰最强。

如果复用因子大，比如 `N=7` 或 `N=12`，同频基站之间距离更远，干扰减弱，但每个小区能使用的频谱资源更少。

这个问题到了低空场景会变得更复杂。因为 UAV 用户升空以后，更容易和多个基站同时形成 LOS 链路，所以服务信号可能增强，干扰也可能同步增强。再考虑基站主瓣下倾、噪声底、站址几何和业务负载后，低空链路质量就不再是一个简单的静态空间问题。

## 三、建模思路
我的整体方法仍然是“解析基线 + Monte Carlo 仿真 + 动态实验”。

第一层是解析模型，用经典六边形蜂窝复用公式给出边缘 SIR 理论基线。复用因子满足

`N = i^2 + ij + j^2`

同频复用距离和小区半径满足

`D / R = sqrt(3N)`

因此经典边缘 SIR 可以写为

`SIR = (3N)^(n/2) / 6`

第二层是静态 Monte Carlo 仿真。这里我不再使用“规则蜂窝 + 全向天线 + 纯路径损耗”的理想化版本，而是加入了：

1. 三扇区宏站方向图；
2. 机械下倾 `4°` 和电下倾 `8°`；
3. 对数正态阴影衰落和 Nakagami 小尺度衰落；
4. 热噪声、带宽和接收机噪声系数；
5. 可选自定义方向图和波束码本。

第三层是动态实验。我接入了一个公开真实站址簇，也就是 Knoxville, Tennessee 周边的真实蜂窝塔点位，并在此基础上引入 UAV 轨迹、时变负载、切换滞回、按用户分功率，以及基于建筑 GIS 的局部 LOS/NLOS 覆盖判决，得到动态 SINR 结果。

## 四、指标定义
当前版本最关键的升级之一，就是从纯 `SIR` 扩展到 `SINR`。

以前的模型只看

`SIR = Signal / Interference`

但当前版本进一步显式加入热噪声，也就是

`SINR = Signal / (Interference + Noise)`

本文带宽取 `20 MHz`，热噪声谱密度取 `-174 dBm/Hz`，接收机噪声系数取 `7 dB`，对应总噪声功率约为 `-93.99 dBm`。

系统级指标方面，我继续采用有效 ASE，而不是理想化的

`ASE = (1/N) log2(1+SIR)`

当前有效 ASE 会同时考虑：

1. 资源活跃度 `0.7`；
2. 调度效率 `0.9`；
3. 控制开销扣减 `1 - 0.18`；
4. `10 dB` 的业务门限。

所以现在的系统指标更接近真实系统能稳定承载的有效业务能力。

## 五、参数与真实数据来源
主要参数包括：

1. 小区半径 `500 m`；
2. 复用因子 `1, 3, 4, 7, 12`；
3. 基站高度 `25 m`；
4. 发射功率 `46 dBm`；
5. 地面路径损耗指数 `3.8`；
6. LOS / NLOS 路径损耗指数 `2.2 / 3.8`；
7. Monte Carlo 样本数 `4000`；
8. UAV 高度从 `0 m` 扫描到 `300 m`。

真实站址部分，我没有再使用纯统计扰动，而是从公开 ArcGIS Hub 数据集 `Cellular Towers in the United States` 中，提取了 Knoxville 周边的一个真实局部站址簇，并转成局部坐标输入动态实验。

这里需要说明的是：这个真实站址簇是公开数据驱动的“真实布局样本”，不是完整商用网络全量复现，但它已经显著提高了空间部署的现实性。

## 六、结果一：复用因子对地面用户 SIR / SINR 的影响
先看地面用户结果。

当路径损耗指数取 `3.8` 时，解析边缘 SIR 从 `N=1` 的 `1.28 dB` 提升到 `N=12` 的 `21.79 dB`。

随机用户中位 SIR 从 `14.25 dB` 提升到 `36.59 dB`。

但如果考虑噪声，中位 SINR 只从 `12.82 dB` 提升到 `20.18 dB`。

这个结果说明一件很重要的事：复用因子增大以后，干扰确实被明显压下来了，但系统并不会无限受益，因为当干扰下降到一定程度后，噪声底会开始限制 SINR 的进一步提升。

所以，当前版本的结论比纯 SIR 更完整：频率复用增大能改善链路，但改善幅度受噪声底约束。

## 七、结果二：复用因子对有效 ASE 的影响
接着看系统级指标。

虽然复用因子增大以后，平均用户链路质量变好，但单位面积有效 ASE 仍然持续下降。

具体来说：

1. `N=1` 时，有效 ASE 为 `3.34 bit/s/Hz/km²`；
2. `N=3` 时，下降到 `1.59 bit/s/Hz/km²`；
3. `N=7` 时，下降到 `0.74 bit/s/Hz/km²`；
4. `N=12` 时，只剩 `0.44 bit/s/Hz/km²`。

这说明系统设计里的经典矛盾依然成立：复用因子越大，干扰越小，但空间复用密度越低。链路更稳，并不代表单位面积总产出更高。

## 八、结果三：低空 SINR 的高度效应
这次低空结果的主指标不再只是 SIR，而是 SINR。

在 `N=7` 条件下，UAV 用户中位 SINR 从地面参考高度 `0 m` 时的 `19.20 dB`，先上升到 `25 m` 时的 `25.98 dB`，然后开始迅速下降：

1. `50 m` 时为 `20.67 dB`；
2. `100 m` 时降到 `1.34 dB`；
3. `200 m` 时降到 `-4.91 dB`；
4. `300 m` 时降到 `-5.59 dB`。

10 dB 门限覆盖概率则从 `0.737` 先升到 `0.793`，再迅速降到 `0.0033`。

这说明当前 SINR 框架下，低空高度效应呈现“近低空短暂受益、高空快速崩塌”的两阶段结构。

## 九、为什么会先升后降
这个结果背后的机理，需要同时看三个量。

第一，服务链路 LOS 概率确实在快速上升。它从 `0 m` 的 `0.105` 提升到 `25 m` 的 `0.402`，再到 `100 m` 的 `0.970`。

第二，服务方向图增益会随着高度升高而恶化。因为基站主瓣是下倾覆盖地面的，用户继续升高以后，会更快偏离主瓣中心。

第三，干扰链路 LOS 概率也会迅速上升，从 `0.0039` 一直升到 `300 m` 时的 `0.713`。

所以低空 SINR 不是简单的“高度越高越差”或者“高度越高越好”，而是在近低空阶段先受到 LOS 收益的支撑，随后被主瓣失配和 LOS 干扰累积压垮。

## 十、结果四：真实站址 + 动态实验
这一版最重要的新内容，是把真实站址和动态业务过程接进来了。

在 Knoxville 的真实站址簇上，我构造了一条 `60 s` 的 UAV 轨迹，并引入时变负载、L3 测量滤波与 Time-to-Trigger 切换事件、主干扰站联合调度近似、按用户分功率，以及覆盖当前站址簇的建筑 GIS 遮挡判决和附加损耗近似。结果如下：

1. 总共发生了 `2` 次切换；
2. 平均 SINR 为 `5.63 dB`；
3. 5% 分位 SINR 低到 `-6.35 dB`；
4. `10 dB` 门限失效概率下降到 `0.700`。

这说明一个非常重要的问题：即使静态中位结果看起来还能接受，真实站址几何和动态业务过程也可能让低空链路在时间轴上变得非常不稳定。

所以低空用户面对的不只是“空间上的干扰更强”，还有“时间上的切换和负载波动更剧烈”。

## 十一、工程启示
从当前分支的结果出发，我认为有三点工程启示。

第一，低空融合网络不能只看静态地面覆盖，更不能只看理想化的 SIR 曲线，必须同时考虑 SINR、站址几何和业务时序。

第二，如果系统目标偏向可靠性和覆盖，尤其是要保障空中终端，那么应该采用更保守的复用策略，并结合更精细的波束配置和干扰协调。

第三，如果系统目标偏向单位面积吞吐量，那么较小复用因子仍然有优势，但必须同时做好功控、调度和低空用户专门资源管理，否则动态过程中的实际体验会明显恶化。

## 十二、剩余局限
虽然当前版本比主分支已经真实得多，但仍有几点需要说明。

1. 真实站址簇已经可以通过公开 ArcGIS 数据脚本化提取，但默认样例仍只是 Knoxville 的局部样本，不是某个运营商完整商用网的全量布局；
2. 自定义方向图接口已经具备，默认方向图也已切换到常见 MSI 设备文件格式，但仍不是厂商级实测方向图；
3. 动态业务过程已经加入轨迹、负载、L3 测量滤波与 Time-to-Trigger 切换事件，也补入了主干扰站联合调度近似，但还没有完整纳入 MIMO 码本和真实商用功控策略；
4. 动态真实站址实验已经可以接入覆盖当前站址簇的建筑 GIS 做确定性遮挡判决，并进一步加入建筑遮挡附加损耗近似，但默认建筑样例仍不是完整城市 3D GIS，也没有严格射线追踪。

这些局限不会改变本文的主结论，但说明当前分支更适合“工程化机理分析”和“方法验证”，而不是直接商用复盘。

## 十三、总结
最后总结一下。

第一，复用因子增大，仍然会显著改善地面用户的 SIR 和 SINR，但 SINR 提升会受到噪声底限制。

第二，在有效 ASE 指标下，单位面积频谱效率会随复用因子增大而下降，所以频率规划本质上仍然是容量和抗干扰能力之间的折中。

第三，低空 UAV 用户的 SINR 随高度呈现“近低空受益、高空崩塌”的两阶段结构，背后的核心机理是服务链路 LOS 增益、主瓣失配和 LOS 干扰累积之间的竞争。

第四，一旦接入真实站址和动态业务过程，低空链路的时序不稳定性会变得非常明显，说明真实网络中的低空问题不仅是空间问题，也是时间问题。

我的汇报到这里结束，谢谢各位老师。

## 十四、常见追问与回答

### 1. 为什么这次一定要用 SINR，而不是继续用 SIR？
回答思路：
因为当前版本已经把系统从纯干扰主导假设推进到了更接近工程实际的绝对功率模型里。加入热噪声以后，可以看到当复用因子继续增大时，SIR 虽然还在变好，但 SINR 改善会逐渐饱和，这一点纯 SIR 是看不出来的。

### 2. 为什么 25 m 会先变好，而不是像主分支那样直接退化？
回答思路：
因为当前分支引入了峰值面板增益、绝对功率标定和噪声底以后，近低空阶段的服务 LOS 收益在一开始还能压住噪声和干扰，所以 SINR 会先上升；但随着高度继续增加，主瓣失配和 LOS 干扰累积开始主导，系统很快转入退化阶段。

### 3. 真实站址动态实验为什么比静态实验差这么多？
回答思路：
因为真实站址几何不再规则，动态过程中又加入了负载波动、按用户分功率和切换逻辑。这样一来，某些时间步会出现强干扰、功率摊薄和切换不稳定的叠加效应，所以时间轴上的极端劣化会比静态平均结果严重得多。

### 4. 当前真实站址接入能不能算“商用网络复现”？
回答思路：
还不能。现在做到的是“公开真实站址簇 + 局部建筑 GIS 驱动的工程化实验”，已经明显优于纯统计扰动模型，但还没有纳入某运营商的完整站址、真实参数、全城级 3D GIS、功控和调度策略，所以更准确的定位是“真实空间样本驱动的机制验证”。

### 5. 如果继续扩展，这个分支最值得往哪里深入？
回答思路：
最值得继续做的有三条：一是把当前公开站址和局部建筑数据扩展到更完整的城市范围；二是把当前波束接口接上更真实的厂商方向图或更强的码本逻辑；三是把动态业务过程进一步扩展到 MIMO、功控和跨小区联合调度。这样就可以从课程设计级模型继续推进到更接近真实网络验证的框架。
